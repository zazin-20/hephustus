# Product Manager

Navigation page for the Product Manager role.

The PM owns **scope, audits, open decisions, and backlog thinking** — the product
framing that feeds the Architect. It decides *what* and *why*; the Architect
decides *how*.

## Contents

| Path | Holds |
|---|---|
| `claude.md` | The Product Manager directive |
| `readme.md` | PM decisions / product framing (referenced by the pipeline) |
| `audit/` | Product and experience audits |
| `system-context/` | Context the PM maintains about the product surface |
| `todo/` | Backlog thinking and open items |
| `admin-comms/` | Stakeholder / admin communications |
| `transcripts/` | Source conversations behind decisions |

## Boundary

- PM owns scope and framing. **Final PRDs are stored by the Architect** in
  `../architect/prds/`.
- PM does not design systems, write issue specs, implement, or verify.
